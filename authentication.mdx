---

All requests use a Bearer token in the `Authorization` header:

```bash
curl "$SMARTSCOUT_API_BASE/api/v1/plugins" \
  -H "Authorization: Bearer $SMARTSCOUT_API_KEY"
```

You can also pass `?token=YOUR_API_KEY` on GET requests when headers are inconvenient (scripts, browsers). Prefer the header in production.

## Credentials you receive

| Item | Example | Purpose |
| ---- | ------- | ------- |
| Base URL | `https://api.example.com` | Host for REST + MCP |
| API key | `ssk_live_...` | Auth for every call |

One key covers both:

- **REST** — `POST /api/v1/call`, `/api/v1/batch`, discovery endpoints
- **MCP** — same key for MCP clients (Cursor, Claude, custom agents)

## Calling tools

```bash
curl -X POST "$SMARTSCOUT_API_BASE/api/v1/call" \
  -H "Authorization: Bearer $SMARTSCOUT_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "tool": "smartscout_get_brand",
    "params": { "brand_id": 12345, "marketplace": "US" }
  }'
```

| Field | Required | Description |
| ----- | -------- | ----------- |
| `tool` | Yes | Full tool name, e.g. `smartscout_search_brands` |
| `params` | No | Tool arguments (object) |
| `account` | Rarely | Only if your key has multiple SmartScout accounts |

## Discovery endpoints

| Method | Path | Description |
| ------ | ---- | ----------- |
| `GET` | `/api/v1/plugins` | List available plugins and accounts |
| `GET` | `/api/v1/tools?q=brand` | Search tools by keyword |
| `GET` | `/api/v1/tools/{tool_name}` | Full parameter schema for one tool |
| `POST` | `/api/v1/call` | Execute one tool |
| `POST` | `/api/v1/batch` | Execute up to 50 tools in one request |

## Failure modes

Unauthorized or out-of-scope tools typically return **404** (endpoint appears missing) rather than leaking existence. Invalid JSON returns **400**.

<Warning>
If a key leaks, rotate it immediately with your account contact. Treat keys like passwords.
</Warning>
