All REST requests use a Bearer token in the `Authorization` header:

```bash
curl "https://smartscout-mcp.fly.dev/api/v1/plugins" \
  -H "Authorization: Bearer $SMARTSCOUT_API_KEY"
```

For MCP clients, pass the same key as `?token=` on the MCP URL (see [MCP setup](/mcp-setup)). Some clients also accept `Authorization: Bearer`. Prefer the header for REST.

## Credentials

| Item | Value | Purpose |
| ---- | ----- | ------- |
| Base URL | `https://smartscout-mcp.fly.dev` | Host for REST + MCP |
| API key | Issued privately (`ssk_...`) | Auth for every call |
| MCP URL | `https://smartscout-mcp.fly.dev/mcp?token=$SMARTSCOUT_API_KEY` | MCP clients |

One key covers both:

- **REST** — `POST /api/v1/call`, `/api/v1/batch`, discovery endpoints
- **MCP** — same key for Cursor, Claude, Codex, and other MCP clients

Keys are issued privately with your plan. Do not publish production keys in docs, repos, or client-side apps.

## Calling tools (REST)

```bash
curl -X POST "https://smartscout-mcp.fly.dev/api/v1/call" \
  -H "Authorization: Bearer $SMARTSCOUT_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "tool": "smartscout_get_brand",
    "params": { "brand_id": 12345, "marketplace": "US" }
  }'
```

| Field | Required | Description |
| ----- | -------- | ----------- |
| `tool` | Yes | Full tool name, e.g. `smartscout_search_brands` |
| `params` | No | Tool arguments (object) |
| `account` | Rarely | Only if your key has multiple SmartScout accounts |

## Discovery endpoints

| Method | Path | Description |
| ------ | ---- | ----------- |
| `GET` | `/api/v1/plugins` | List available plugins and accounts |
| `GET` | `/api/v1/tools?q=brand` | Search tools by keyword |
| `GET` | `/api/v1/tools/{tool_name}` | Full parameter schema for one tool |
| `POST` | `/api/v1/call` | Execute one tool |
| `POST` | `/api/v1/batch` | Execute up to 50 tools in one request |

## Rate limits

API keys can be rate-limited by plan (for example, a daily request cap such as 100 requests/day). Limits are enforced per key.

## Failure modes

Unauthorized or out-of-scope tools typically return **404** (endpoint appears missing) rather than leaking existence. Invalid JSON returns **400**.

<Warning>
If a key leaks, rotate it immediately. Treat keys like passwords.
</Warning>
